# 爬虫原理：请求与响应解析

本文档聚焦爬虫在发送请求与解析响应时的核心原理与实务要点，适合作为工程化爬虫的参考。

## 一、HTTP 请求与响应概览
- 流程：DNS -> TCP（三次握手）-> TLS（HTTPS）-> 发送 HTTP 请求 -> 服务端处理 -> 返回 HTTP 响应 -> 连接复用/关闭。
- 主要组成：
  - 请求行：方法（GET/POST/...）+ URL + HTTP 版本
  - 请求头：User-Agent、Accept、Referer、Cookie、Authorization 等
  - 请求体：表单、JSON、二进制
  - 响应状态码：2xx/3xx/4xx/5xx；响应头（Content-Type、Set-Cookie、Content-Encoding 等）；响应体（HTML/JSON/XML/二进制）

## 二、请求构造与发送策略
- 使用连接复用（session/persistent connection）减少握手开销。
- 必要请求头：合理设置 User-Agent、Accept、Referer、Accept-Language、Accept-Encoding。
- 参数放置：GET -> query string；POST -> body（application/x-www-form-urlencoded 或 application/json）。
- 超时与重试：分别设置 connect/read 超时，网络/临时错误采用指数退避重试（例如 3 次，间隔 1s、2s、4s）。
- 代理与会话：
  - 使用代理池避免单 IP 被封；区分透明/匿名/高匿代理。
  - 使用 cookie jar 或登录会话管理（requests.Session、浏览器上下文）。
- 并发控制：限制并发请求数与并发域（同域并发），避免对目标站点造成压力。

## 三、响应解析（编码、压缩、类型）
- 解码：优先使用响应头 charset，若缺失使用内容嗅探（apparent_encoding）。
- 压缩处理：支持 gzip、deflate。
- 内容类型：
  - HTML：使用解析器（lxml、BeautifulSoup）进行 DOM 抽取。
  - JSON：直接解析为结构化数据（resp.json()）。
  - XML：使用 lxml.etree 或 xml 模块解析。
  - 二进制（图片/文件）：按流保存。

## 四、HTML 抽取实用方法
- 选择器：
  - CSS 选择器（BeautifulSoup.select）便捷。
  - XPath（lxml）更精确且性能通常更好。
- 避免用正则直接解析 HTML（复杂易破碎），正则可用于清洗文本或解析简单结构。
- 提取模式：
  - 先定位列表页的 item 列表，再抓详情页。
  - 使用结构化字段（JSON-LD、microdata）优先获取数据（在 <script type="application/ld+json"> 中）。
- 清洗：去除多余空白、HTML 实体、非必要标签；统一时间/数值格式。

## 五、JavaScript 渲染页面处理
- 优先策略：通过浏览器 DevTools 的 Network 面板寻找 XHR/API 接口，直接请求后端 API 更高效稳定。
- 若必须渲染：
  - 无头浏览器：Playwright、Selenium、Puppeteer（适合复杂 JS）。
  - 轻量渲染：使用 headless 浏览器只渲染必要 DOM。
  - 服务化渲染：Splash 或专门的渲染层。
- 注意：无头浏览器性能和资源消耗高，应限制并发与复用浏览器上下文。

## 六、并发、异步与扩展
- 同步库（requests）适合小规模、调试和简单任务。
- 异步库（aiohttp + asyncio）适合高并发（配合限速、并发信号量、任务队列）。
- 分布式：任务队列（RabbitMQ、Redis Queue）、调度器（Scrapy、自建调度）用于水平扩展。
- 去重与断点续抓：使用唯一键/URL 哈希、布隆过滤器、数据库唯一索引记录抓取状态。

## 七、错误处理与稳定性
- 常见失败：超时、连接重置、403/429（限制）、Captcha、结构变化导致解析失败。
- 处理策略：
  - 分类重试（网络错误/5xx -> 重试；4xx -> 不重试或调整策略）。
  - 切换代理或延长等待时间。
  - 增加健壮的解析容错（try/except、fallback 选择器）。
  - 日志和监控（失败率、响应时间、错误样式）及告警配置。

## 八、防护、礼貌与合规
- robots.txt：优先检查 robots.txt 并遵守（尤其对抓取频率和路径限制）。
- 限速与并发：对同站设定合理的速率限制（如 1-5 秒/请求或更保守），并发控制同域并发数。
- 法律合规：遵守目标站点使用条款和当地法律，不绕过验证码或访问受限/隐私数据。
- 反爬对策：遇到复杂反爬（指纹、行为检测、验证码）时评估合法性并考虑与网站沟通或放弃。

## 九、常用工具与库（速览）
- HTTP 客户端：requests、httpx、aiohttp
- 解析：lxml、BeautifulSoup、parsel（Scrapy 的选择器）
- 无头浏览器：Playwright、Selenium、Puppeteer
- 调度/框架：Scrapy、Frameless 自建队列、Celery/Kafka
- 去重/缓存：Redis、布隆过滤器、SQLite/Postgres（持久化）
- 代理池：自建或第三方服务

### requests + BeautifulSoup（同步）
```python
# filepath: c:\Users\tianr\Desktop\my_spiders\examples\sync_example.py
import requests
from bs4 import BeautifulSoup
from time import sleep

headers = {'User-Agent': 'Mozilla/5.0 (compatible)'}
with requests.Session() as s:
    resp = s.get('https://example.com/list', headers=headers, timeout=(3,10))
    resp.encoding = resp.apparent_encoding
    if resp.status_code == 200:
        soup = BeautifulSoup(resp.text, 'lxml')
        for item in soup.select('.item'):
            title = item.select_one('.title').get_text(strip=True)
            link = item.select_one('a')['href']
            print(title, link)
            sleep(0.5)  # 礼貌限速
```

## 十一、实践建议（工程化）
- 开发流程：本地开发 -> 测试站/沙箱 -> 小流量试运行 -> 扩容部署。
- 配置项化：将 headers、代理、速率、重试策略配置化，方便线上调整。
- 数据保全：抓取前备份、使用事务或幂等写入保证数据一致性。
- 监控与报警：抓取成功率、失败样本、响应时延等指标报警。
